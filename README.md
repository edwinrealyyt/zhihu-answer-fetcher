# 🛠️ 知乎控制器 (Zhihu Answer & Comment Fetcher)

知乎控制器是一款专门为知乎网页端打造的浏览器用户脚本（Userscript）。它能够帮助你一键获取、去重并导出问题下的**所有回答**与**完整评论**。无论你是进行舆情分析、学术研究，还是纯粹的深度阅读，它都能极大提升你的内容获取效率。

---

## 🌟 核心特色与主打功能

### 1. 🔓 强制解除限制，自由多维度排序
* **绕过官方隐藏**：知乎官方出于各种策略，在部分敏感、时政或特定的提问下，会**强行隐藏“按时间排序”**的按钮。
* **任意问题强制排序**：本脚本通过直接调用底层 API 接口拉取最原始的数据，并在脚本端重构排序算法。**在任何提问下，你均可强行启用“点赞数降序”、“时间倒序（最新）”、“时间正序（最早）”**。

### 2. ⚡ 双通道高并发，极速全量加载
* **告别卡顿的瀑布流**：知乎官方的滚动瀑布流加载效率低下，当有成百上千条回答时，频繁滚动极易导致浏览器卡死、内存泄露。
* **高并发挂机爬取**：脚本采用**双通道高并发直连 API 抓取技术**，并配有**智能看门狗异常重试/唤醒机制**。数十秒内即可静默抓取成百上千条数据。
* **后台静默抓取**：支持在后台标签页挂机运行，抓取完毕后会自动提醒，无需守在页面前反复滚动。

### 3. 💬 完美重建评论区，二级评论全量自动补全
* **二级评论无遗漏**：知乎自带的评论区经常会出现折叠、深层子回复截断的情况。本脚本在点击查看评论时，会**在后台自动静默递归补全二级、三级等全部嵌套子回复**。
* **评论多维度排序**：支持评论按“🔥默认”、“👍点赞数”、“💬回复数”、“最新（时间倒序）”和“最早（时间正序）”自由切换排序。
* **解决被删/折叠漏看问题**：如果原始主评论被系统删除或折叠，脚本会智能生成虚拟占位节点，将“无家可归”的深层子评论整齐地缩进挂载，保证上下文逻辑链条完整。

### 4. 🎨 精致现代的悬浮控制面板
* **自由拖拽与位置记忆**：面板标题栏支持按住自由拖动，且会自动在本地 `localStorage` 记忆位置，每次打开页面都会停留在你最顺手的地方。
* **暗黑/浅色双配色记忆**：控制面板支持一键切换主题，内建科技感暗黑模式与极简主义白底浅色模式，同样支持本地首选项记忆。
* **新标签页打开保护**：评论区的所有图片和用户链接默认全部在**新标签页中打开**，彻底防止因不小心误触导致页面在本页跳转、前功尽弃丢失已抓取的回答。

## 🔧 安装方式

1. 安装浏览器脚本管理器插件：推荐在 Chrome / Edge / Firefox 等浏览器中安装 [Tampermonkey (油猴)](https://www.tampermonkey.net/)。
2. 安装本脚本：
   * **方法一**：访问 GreasyFork 搜索“知乎控制器”一键安装。
   * **方法二**：新建用户脚本，将本仓库 `zhihu-answer-fetcher.user.js` 代码复制进去并保存。
3. 打开任意知乎问题页面（URL 匹配 `zhihu.com/question/*`），即可在页面右侧看到浮动的控制面板。

## ❓ 常见使用问答 (FAQ)

### Q1: 为什么脚本截获的“实际回答数量”和知乎页面顶部显示的“回答总数”对不上？
**A**: 这是正常且非常普遍的现象，主要有以下几个深层技术原因：
1. **数据缓存与虚高**：知乎页面顶部的“回答总数”是一个缓存统计计数器。哪怕有用户删除了回答，或者回答被系统管理员封禁折叠，这个计数器**通常不会实时减少**，导致显示出的数字往往比真实存在的回答要多。
2. **敏感与折叠过滤**：部分违规、敏感或处于草稿状态的回答，无法被公共数据接口拉取到。
3. **分页去重过滤**：知乎官方接口在翻页抓取时经常会吐出**重复的回答数据**。本脚本在接收到数据后，通过唯一的 `Answer ID` 进行了一次严格去重，确保你看到的每一条都是真实唯一的，这也会使得计数看起来有所减少。

### Q2: 挂机抓取时切到其他网页，为什么切回来后有时候显示有延迟？
**A**: 现代浏览器（如 Chrome、Edge）为了节省系统资源，会对后台非活跃标签页的定时器（`setTimeout`）进行非常严格的限频与挂起。
脚本针对此特性进行了深度优化：
1. **轻能耗心跳自循环**：在后台无阻塞时以 1 秒/次的频率轻量自我检查。
2. **瞬时唤醒机制**：脚本监听了网页的可见性变化（`visibilitychange`）。一旦您从其他网页**切回知乎页面**，脚本会在 50 毫秒内瞬间激活检测并刷新界面状态。

### Q3: 为什么部分用户的名字点击跳转链接是 `/people/{id}`（一长串哈希值）？
**A**: 在知乎中，部分用户没有手动设定过个性的网址域名（即 `url_token` 缺失）。针对这部分用户，脚本会自动回退提取其唯一的系统物理 `id`。
**请放心使用**：这属于知乎官方支持的标准短链格式，当您点击链接时，知乎服务器会自动识别此 ID 并安全重定向至该用户的个人真实主页。
